## 역사와 특징
- Linux는 1991년 리누스 토르발스(Linus Torvalds)가 개발한 Unix 계열의 오픈소스 커널[^kernel]이다.
- 1991년 10월 5일 Linux 커널 0.02 버전이 공개되면서 알려지기 시작했다.
- 초기부터 멀티유저(Multi-user)와 멀티태스킹(Multi-tasking) 환경을 지원했다.
- 이후 멀티프로세서 환경을 지원하도록 발전했다.
- Unix 계열과 유사한 계층형 파일 시스템을 사용하며, POSIX 표준을 상당 부분 지원하여 Unix 계열 환경과 높은 호환성을 가진다.

## 철학과 라이선스
- GNU: 'GNU는 유닉스가 아니다(GNU's Not Unix)'의 약자
- FSF: 1985년 리처드 스톨만이 설립한 재단(Free Software Foundation)
- OSS: 소스 코드를 공개하여 사용·수정·배포할 수 있도록 한 소프트웨어를 의미한다.
   - 자유 소프트웨어와 비슷한 가치를 공유하지만, 자유 소프트웨어는 사용자의 자유를, OSS는 개발 방식과 실용성을 더 강조한다.
- GPL: FSF에서 만든 대표적인 자유 소프트웨어 라이선스.
   - GPL 코드를 수정하거나 결합한 프로그램을 배포할 때는, 해당 소스 코드도 GPL 조건으로 공개해야 한다(Copyleft).[^copyleft]
- LGPL: GPL보다 훨씬 완화된 조건의 자유 소프트웨어 라이선스
   - LGPL이 적용된 라이브러리를 이용하여 개발했을 경우 프로그램 소스코드를 공개하지 않아도 됨. 하지만 LGPL라이브러리 자체를 수정했다면 수정한 라이브러리 부분의
     소스 코드는 LGPL 조건[^lgpl]에 따라 공개해야 한다.
     간단한 예시로

     내가 A라는 LGPL 라이브러리를 가져와서 회원 관리 프로그램을 만들었다고 하자.
     A 라이브러리를 그대로 사용하고, 내 회원 관리 프로그램만 작성함
     → 내 프로그램의 소스 코드를 공개하지 않고 판매·배포할 수 있음.
     A 라이브러리의 기능을 직접 수정함
     → 수정한 A 라이브러리 부분의 소스 코드는 LGPL 조건에 따라 공개해야 함.
     → 단, 내 회원 관리 프로그램 전체를 공개할 필요까지는 없음.

## 대표적인 리눅스 배포판  
#### 강의 내용에는 오래된 교재, 시험 자료 기준의 설명이 포함되어 있는것 같아 구분하여 정리함

- Red Hat Linux
  - 강의: 미국의 Red Hat사가 개발하던 리눅스 배포판이다.
  - 현재: Red Hat Linux는 2004년에 개발이 종료된 과거의 배포판이다. 이후 Red Hat 계열은 Fedora와 RHEL을 중심으로 이어졌다.[^redhat]
- Ubuntu
  - 강의: Debian GNU/Linux에 기초한 운영체제이다.
  - 현재: Debian 기반의 대표적인 리눅스 배포판이다. 데스크톱, 서버, 클라우드 환경에서 널리 사용된다.
- SUSE Linux
  - 강의: 독일에서 출시된 배포판으로 유럽에서 인기가 많다.
  - 현재: 독일에서 시작된 리눅스 배포판 계열이다.
- Debian
  - 강의: Debian 프로젝트에서 만들어 배포하는 공개 운영체제이며, GNU의 공식적인 후원을 받는 유일한 배포판이다.
  - 현재: Debian 프로젝트가 개발·배포하는 자유·오픈 소스 리눅스 배포판이다. 안정성과 APT 패키지 관리 시스템이 특징이며, Ubuntu를 포함한 여러 배포판의 기반이 된다.
- RHEL(Red Hat Enterprise Linux)
  - 강의: Red Hat사가 개발하여 판매하는 상용 리눅스 배포판이다.
  - 현재: Red Hat이 개발·지원하는 기업용 상용 리눅스 배포판이다. 장기 지원과 기술 지원을 제공하며, 서버·기업 환경에서 많이 사용된다.
- CentOS
  - 강의: RHEL 업스트림 소스를 기반으로 하며, RHEL과 완벽하게 호환되는 무료 기업용 운영체제이다.
  - 현재: 기존 CentOS Linux는 2021년 12월 31일 지원이 종료되었다. 현재 CentOS Stream은 RHEL보다 앞선 개발 단계에 가까운 배포판이다. 따라서 현재의 CentOS Stream을
    "RHEL과 완벽하게 호환되는 무료 복제판"으로 설명하면 부정확하다.[^centos]
- Slackware Linux
  - 강의: 초기 리눅스 배포판 가운데 가장 먼저 대중화된 배포판이다.
  - 현재: 1993년 패트릭 볼커딩(Patrick Volkerding)이 처음 공개한 초기 리눅스 배포판 중 하나이다. 단순성과 전통적인 Unix 방식을 중시한다.
- Fedora Linux
  - 강의: 리눅스 커널에 기반한 운영체제이며, Red Hat의 후원과 개발 공동체의 지원 아래 개발된 배포판이다.
  - 현재: Red Hat의 후원과 커뮤니티의 참여로 개발되는 배포판이다. 최신 기술을 비교적 빠르게 도입하며, RHEL 개발에 앞서 새로운 기술을 검증하는 업스트림 성격의 배포판이다.

## 리눅스의 구조 및 장점
- 운영체제의 역할
 - 사용자로 하여금 하드웨어를 컨트롤할 수 있도록 중간다리 역할 수행.
 - 리눅스에서는 사용자가 명령어 입력 -> 셸(Shell)[^shell]이 명령어를 해석하여 커널(Kernel)로 전달 -> 커널이 요청된 프로그램이 실행될 수 있도록 처리한다.


[^kernel]: (../Terms#Kernel) 
[^copyleft]: GPL 코드를 활용해 배포한다면, 받은 사람도 수정·공유할 자유를 가져야 한다
[^lgpl]: LGPL조건은 GPL보다 완화된 Copyleft 규칙으로 LGPL 라이브러리를 사용한 내 프로그램은 비공개로 배포할 수 있지만, LGPL 라이브러리 자체를 수정했다면 수정한 
          라이브러리 부분은 공개해야 한다. 
[^redhat]: 과거 Red Hat Linux의 일반 사용자용 성격은 Fedora가 이어받았고, 기업용 상용 배포판은 RHEL이 담당하게 되었다.
[^centos]: 과거의 CentOS Linux는 RHEL 소스 기반의 무료 호환 배포판이었다. 현재 CentOS Stream은 Fedora와 RHEL 사이에서 RHEL의 다음 릴리스 개발에 가까운 위치를 가진다.
[^shell]: 셸은 사용자와 커널 사이에서 명령을 해석하고 실행을 요청하는 명령어 해석기 프로그램이다.
