## kernel

커널(Kernel)은 운영체제의 매우 중요한 핵심 부분이다. 하드웨어 자원을 필요한 프로세스에 나눠주고 프로세스 관리, 메모리 제어 등을 수행한다.
예를 들어 웹 브라우저와 VS Code, 음악 프로그램을 동시에 실행하면 커널은 각 프로그램에 CPU 사용 시간과 메모리를 나누어 배정한다.


## Shell

셸은 사용자와 커널 사이에서 명령을 해석하고 실행을 요청하는 명령어 해석기 프로그램이다.


## Subnet Mask

서브넷 마스크란 IP 주소에서 네트워크 ID와 호스트 ID를 구분하기 위해 사용하는 32비트 이진수 비트 마스크이다. 동작 방식은 자신의 IP 주소와 서브넷 마스크를 비트 단위 AND 연산하여 자신이 속한 네트워크 주소를 도출한다.
<br><br>ex) 내 IP 주소가 192.168.0.10라고 하면 이진수로 변환해보면 11000000.10101000.00000000.00001010이고 서브넷 마스크 255.255.255.0을 이진수로 변환하면 11111111.11111111.11111111.00000000 AND연산 해보면 네트워크 ID 11000000.10101000.00000000.00000000 -> 192.168.0.0이 나온다.<br><br>
+ IP 주소에서 앞에 3개 옥텟(8비트 대역)을 네트워크 ID, 제4 옥텟을 호스트 ID로 구분할 수 있는데(Classful C) 굳이 서브넷 마스크를 사용해야 하는 이유로
  #### 첫번째:
  조직 규모에 맞게 네트워크를 쪼갤 수 있다(서브네팅).서브넷 마스크가 없다면 무조건 규칙대로 앞의 3칸(192.168.0)을 네트워크 ID로 써야한다. 이 경우, 마지막 옥텟(0~255)을 통해 총 256개의 주소를 강제로 할당받게 되는데 이렇게 되면 30대만 쓰고 남은 226개의 주소는 아무도 못 쓰고 버려진다. 서브넷 마스크가 있다면 서브넷 마스크 값을 조절하여(CIDR) 앞의 3개 옥텟만 아니라 마지막 옥텟의 일부까지도 네트워크 ID로 묶어버릴 수 있어 낭비를 줄일 수 있다.<br> 
  #### 두번째:
   컴퓨터가 목적지가 같은 네트워크에 있는지 판단할 수 있다. 컴퓨터는 자신의 IP 주소와 서브넷 마스크를 AND 연산하여 자신의 네트워크 주소를 구하고, 목적지 IP 주소에도 같은 서브넷 마스크를 적용하여 목적지 네트워크 주소를 구한다. 두 네트워크 주소가 같다면 같은 네트워크에 있다고 판단하여 직접 전송하고, 두 네트워크 주소가 다르면 다른 네트워크에 있다고 판단하여 기본 게이트웨이로 전송한다.<br><br>
+ /24 네트워크는 전체 주소는 256개지만, 일반적인 호스트에 할당 가능한 주소는 네트워크 주소와 브로드캐스트 주소를 제외한 254개다.<br>
  * 브로트캐스트 주소: 같은 서브넷에 속한 모든 호스트에게 데이터를 한 번에 전송하기 위한 IPv4 주소이다. 서브넷 마스크를 기준으로 했을 때, 호스트 ID 부분의 모든 비트를 1로 설정한 주소다. <br><br>ex) 네트워크 192.168.0.0/27, 서브넷 마스크 255.255.255.224 (/27네트워크 에서는 마지막 5비트가 호스트ID)이면 호스트 ID 부분 비트를 1로 설정했을 때 11111.
    십진수로 변환하면 31. 따라서 브로드캐스트 주소는 192.168.0.31이 된다. 사용 가능한 네트워크 수는 192.168.0.0, 192.168.0.31를 제외한 192.168.0.1~192.168.0.30까지 30개.


## Gateway

게이트웨이(Gateway)는 서로 다른 네트워크 사이에서 데이터를 전달하는 출입구 역할을 한다.<br><br>
같은 네트워크에 있는 장치끼리는 일반적으로 게이트웨이를 거치지 않고 직접 통신하지만, 인터넷처럼 다른 네트워크에 있는 장치와 통신하려면 기본 게이트웨이(Default Gateway)를 통해 데이터를 전송한다.<br><br>
일반적으로 가정이나 학교, 회사 네트워크에서는 공유기 또는 라우터의 IP 주소가 기본 게이트웨이로 설정되고, 내 컴퓨터가 같은 네트워크에 없는 외부 인터넷 서버로 데이터를 보낼 때, 데이터는 먼저 기본 게이트웨이로 전달된다.<br><br>
가정에서 사용하는 공유기는 보통 NAT(Network Address Translation) 주소 변환 기능도 수행한다. 192.168.0.10과 같은 사설 IP 주소는 인터넷에서 직접 사용할 수 없으므로, 공유기는 데이터를 외부로 보낼 때 사설 IP 주소를 공인 IP 주소로 변환한다. 외부 웹 서버의 응답이 돌아오면 공유기는 변환 정보를 참고하여 응답 데이터를 다시 192.168.0.10으로 전달한다.<br><br>
ex) 내 컴퓨터의 IP 주소가 192.168.0.10이고 기본 게이트웨이가 192.168.0.1이라고 하자. 
- 목적지 IP 주소가 192.168.0.20인 경우
 - 내 컴퓨터와 같은 네트워크에 속해 있으므로, 기본 게이트웨이를 거치지 않고 192.168.0.20에 직접 데이터를 전송한다.
- 목적지 IP 주소가 외부 웹 서버인 경우
 - 외부 웹 서버는 현재 네트워크에 속하지 않으므로, 내 컴퓨터는 패킷을 기본 게이트웨이인 192.168.0.1로 전송한다.<br><br>
기본 게이트웨이는 패킷을 받으면 다음과 같이 동작한다.<br>

1. 패킷에 포함된 목적지 IP 주소를 확인한다.
2. 라우팅 테이블(Routing Table)을 확인한다.
3. 목적지까지 가기 위해 패킷을 어느 방향으로 보낼지 결정한다.
4. 인터넷에 있는 목적지라면 인터넷 회선 또는 다음 라우터로 패킷을 전달한다.
5. 목적지 서버의 응답 패킷이 돌아오면, 요청을 보낸 내부 컴퓨터에 다시 전달한다.


## Router

라우터는 서로 다른 네트워크 사이에서 패킷을 전달하는 네트워크 장비(노드)이다.<br><br> 목적지 IP 주소와 라우팅 테이블을 참고하여 패킷을 어느 방향으로 보낼지 결정한다.
* 라우팅 테이블(Routing Table): 목적지 네트워크별로 전달받은 패킷을 어디로 보내야 하는지 기록한 경로 정보이다.<br>
* 기본 경로(Default Route): 라우팅 테이블에 목적지로 가는 구체적인 경로가 없을 때 사용하는 기본 경로이고, 일반적으로 외부 인터넷으로 가는 패킷은 기본 경로에 따라 기본 게이트웨이로 전송된다.<br><br>
  + 게이트웨이가 라우터로 패킷을 전송하는 역할이 있다고 했는데 그럼 무한 뺑뺑이를 돌리는 없는데 뭘까? A) 내 PC에서 디폴트 라우트만 가지고 기본 게이트웨이(가정용 공유기)로 전송
  -> 외부 네트워크 경로는 모르니 상위 라우터(ex 통신사)로 넘김 -> 구체적인 경로로 패킷 전송.<br>
  + 만약 어떠한 에러가 나서 뺑뺑이를 돈다 해도 이를 방지하기 위해 IP 패킷에는 TTL(Time To Live)이라는 숫자가 적혀 있는데(일반적으로 운영체제에 따라 64, 128 등) 라우터를 하나 거칠 때마다 TTL 수치가 1씩 감소하고, 수치가 0이 되는 순간 라우터는 해당 패킷을 즉시 폐기하고 송신자에게 오류 메시지를 전송한다

    
## Routing

라우팅(Routing)은 데이터 패킷이 목적지 네트워크까지 이동할 경로를 결정하고, 해당 경로로 패킷을 전달하는 과정이다.<br><br>
운영체제와 라우터는 라우팅 테이블(Routing Table)을 참고하여 목적지 IP 주소에 따라 패킷을 직접 전송할지, 특정 네트워크 인터페이스로 전송할지, 게이트웨이로 전송할지를 결정한다.
+ 네트워크 인터페이스(Network Interface): 컴퓨터나 라우터가 네트워크에 연결되고 데이터를 송수신하는 통로이다. 유선 랜카드, 무선 랜카드, 가상 네트워크 어댑터 등이
  네트워크 인터페이스에 해당하며, Linux에서는 eth0, ens33, wlan0 등의 이름으로 표시될 수 있다.
Linux에서 ip route 명령어를 사용하여 라우팅 테이블과 기본 게이트웨이 정보를 확인할 수 있다.
